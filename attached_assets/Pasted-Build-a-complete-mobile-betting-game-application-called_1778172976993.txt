Build a complete mobile betting game application called "3 Batti" using React Native (Expo) for frontend, Node.js/Express backend deployed on Render, and Neon PostgreSQL database. The app should be compiled into an APK using Expo Go. Admin panel should be a separate React web app built and deployed on Hostinger (dist folder).

## APP NAME & BRANDING
- App Name: "3 Batti" (Three Lights)
- Theme: Traffic light concept - Red, Yellow, Green
- Logo: Traffic signal with 3 circular lights (Red top, Yellow middle, Green bottom) on green background with yellow bottom strip
- App Icon: Same traffic light logo with rounded corners
- Primary Colors: Green (#2E7D32), Yellow (#FBC02D), Red (#D32F2F), Dark Blue/Navy for admin panel (#1A237E)

## GAME CONCEPT (3 BATTI)
This is a color and number betting game. Users bet on which color (Red/Yellow/Green) or number (0-9) will appear in the next round. The game runs 4 times daily at fixed schedules:
- Morning: 9:00 AM
- Afternoon: 1:00 PM (Dopahar)
- Evening: 5:00 PM
- Night: 9:00 PM

Each round has a countdown timer showing time remaining until next game.

## USER APP FEATURES (MOBILE APP - REACT NATIVE EXPO)

### 1. AUTHENTICATION SYSTEM
- Mobile number registration with OTP verification
- Login with MPIN (4-digit) or Biometric (fingerprint/face ID)
- "Remember me" option
- Forgot MPIN reset via OTP
- User Profile: Name, Mobile, Email (optional), Profile Picture
- KYC Status: Pending/Verified (for withdrawals above certain amount)

### 2. WALLET & PAYMENTS
- Wallet Balance Display (prominently shown in header)
- Add Money:
  - Minimum deposit: ₹100
  - UPI ID payment option (user enters UPI ID, gets QR code to scan)
  - Reference/UTR ID mandatory input after payment
  - Upload payment screenshot option
  - Status: Pending → Admin Approval → Credited
- Withdraw Money:
  - Minimum withdrawal: ₹100
  - Add bank details: Account Number, IFSC, Account Holder Name, Bank Name
  - OR UPI ID withdrawal option
  - Withdrawal request → Admin Approval → Processed
- Transaction History:
  - All deposits, withdrawals, bets, winnings, referral bonuses
  - Filter by: All, Credit, Debit, Pending
  - Date-wise sorting

### 3. BETTING SYSTEM (CORE GAME)
- **Home Screen:**
  - Countdown timer to next game (9 AM, 1 PM, 5 PM, 9 PM)
  - 3 large circular buttons: RED, YELLOW, GREEN (traffic light style)
  - Number grid 0-9 below colors
  - "Place Bet" button
  
- **Betting Rules:**
  - Minimum bet: ₹10
  - Color Betting: Choose Red, Yellow, or Green
    - If chosen color wins: 2x return (₹10 bet = ₹20 total)
    - If lose: ₹0 return
  - Number Betting: Choose any number 0-9
    - If chosen number wins: 9x return (₹10 bet = ₹90 total)
    - If lose: ₹0 return
  - User can place multiple bets in one round (different colors/numbers)
  - Show potential winning amount before confirming bet
  
- **Bet Confirmation:**
  - Selected color/number + amount display
  - Confirm/Cancel option
  - Deduct amount from wallet immediately upon confirmation
  - Show "Bet Placed Successfully" animation

### 4. RESULTS & HISTORY
- Live Result Display:
  - Current winning color (full screen animation - glowing effect)
  - Current winning number (large display)
  - "You Won ₹XXX" or "Better Luck Next Time" message
  - Auto-credit winnings to wallet
  
- Result History Table:
  - Date | Time | Winning Color | Winning Number | Your Bet | Result | Amount
  - Last 50 results visible
  - Filter by date range

### 5. REFERRAL SYSTEM
- Unique referral code for each user (auto-generated, customizable)
- Share via WhatsApp, SMS, Copy Link
- Referral Benefits:
  - Referrer gets ₹50 when referee registers and adds ₹100+
  - Referee gets ₹25 signup bonus
  - Lifetime commission: Referrer gets 1% of every bet placed by referee (ongoing)
  - Multi-level: If referee refers someone, original referrer gets 0.5%
- Referral Dashboard:
  - Total referrals
  - Total referral earnings
  - List of referred users with their betting activity
  - Earnings breakdown: Signup bonus + Betting commission

### 6. PROFILE & SETTINGS
- Edit Profile (Name, Email, Profile Pic)
- Change MPIN
- Bank/UPI Details management
- Notification Settings (game reminders, result announcements)
- Dark Mode toggle
- Language: Hindi/English toggle
- Logout

### 7. NOTIFICATIONS
- Push notifications 15 minutes before each game
- Result announcement notification
- Wallet credit/debit notifications
- Referral bonus credit notification
- Admin announcements

### 8. SUPPORT & LEGAL
- Help & Support: WhatsApp chat button, FAQ
- About Us
- Terms & Conditions
- Privacy Policy

## TECHNICAL REQUIREMENTS

### FRONTEND (MOBILE APP)
- React Native with Expo SDK
- Expo Router for navigation
- Redux Toolkit for state management
- React Native Paper/NativeBase for UI components
- Reanimated 2 for animations (betting glow, winning celebration)
- React Native Chart Kit for user stats
- Expo Notifications for push notifications
- AsyncStorage for local data
- Axios for API calls
- expo-image-picker for screenshot upload

### BACKEND (RENDER)
- Node.js with Express.js
- Neon PostgreSQL (serverless Postgres)
- Prisma ORM for database management
- JWT authentication (access + refresh tokens)
- bcrypt for password hashing
- Multer for file uploads (screenshots, QR codes)
- node-cron for game scheduling automation
- Socket.io for real-time bet updates in admin panel
- express-rate-limit for API security
- Helmet for security headers
- CORS configured for specific domains

### DATABASE SCHEMA (NEON POSTGRESQL)
Tables needed:
1. Users (id, name, mobile, email, mpin_hash, wallet_balance, referral_code, referred_by, kyc_status, created_at)
2. Wallets (id, user_id, balance, total_deposited, total_withdrawn, total_won, total_lost)
3. Transactions (id, user_id, type [deposit/withdrawal/bet/win/referral/bonus], amount, status, reference_id, screenshot_url, created_at)
4. Games (id, scheduled_at, status [upcoming/live/closed], winning_color, winning_number, declared_by, total_collection, total_payout, created_at)
5. Bets (id, user_id, game_id, bet_type [color/number], selection [red/yellow/green/0-9], amount, potential_win, status [pending/won/lost], created_at)
6. UPI_Settings (id, upi_id, qr_image_url, is_active, user_range_start, user_range_end, display_order)
7. Admin_Users (id, username, password_hash, role, permissions, last_login, created_at)
8. Audit_Logs (id, admin_id, action, details, ip_address, created_at)
9. Notifications (id, user_id, title, message, type, is_read, created_at)
10. Referral_Earnings (id, referrer_id, referee_id, type [signup/bet], amount, status, created_at)

## IMPORTANT NOTES FOR DEVELOPER
1. The game MUST have manual result declaration by admin as default (not random) so admin can control profitability
2. Admin should see bet distribution BEFORE declaring result to make informed decision
3. UPI rotation system is critical - different users should see different UPI IDs to distribute load
4. All financial transactions must have proper audit trail
5. Implement proper error handling and user-friendly error messages in Hindi/English
6. App should work smoothly on low-end Android devices (2GB RAM)
7. Optimize images and API calls for slow internet (2G/3G compatibility)
8. Add "Responsible Gaming" warnings and self-exclusion option
9. Implement daily betting limits to prevent addiction
10. Keep 10% of total deposits as platform reserve (shown in admin dashboard)

Build this complete system with clean code, proper folder structure, comments, and documentation. Make sure the UI is attractive, modern, and engaging so users enjoy playing. The traffic light theme should be prominent throughout the app with smooth animations.
